Attenzione: Simulazione Didattica ed Etica

Questa pagina è progettata esclusivamente a scopo educativo. Inserisci dati fittizi o simulati per comprendere le meccaniche. Il sistema esegue un'analisi matematica locale sul tuo browser. Nessun dato viene trasmesso, salvato o utilizzato per attacchi reali. Attaccare sistemi senza autorizzazione è un reato penalmente perseguibile.

Brute Force

Tenta ogni possibile combinazione di caratteri. Lento ma matematicamente infallibile se il tempo è infinito.

Dictionary Attack

Usa liste di parole comuni (es. "password123", "admin"). Estremamente veloce contro password prevedibili.

Credential Stuffing

Riutilizza credenziali rubate da altre violazioni (data breach). Funziona se usi la stessa password ovunque.

Dati Account (Simulati)

Come difendersi (Best Practice)

  • Lunghezza > Complessità: Una passphrase ("CavalloBatteriaSpillaCorretta") è meglio di una corta ("P@ss1").
  • Non riutilizzare: Usa una password diversa per ogni servizio per prevenire il credential stuffing.
  • Usa un Password Manager: Strumenti come Bitwarden o 1Password ricordano e generano password complesse per te.
  • Attiva l'autenticazione a due fattori (2FA): Anche se indovinano la password, l'attaccante viene bloccato senza il secondo codice.
